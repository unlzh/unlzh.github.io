使用Hexo搭建GitHub个人博客（一）
2018-11-21|搭建
字数总计: 696|阅读时长: 2 分钟

演示环境

  • Windows 10（Linux下搭建可参考使用Hexo搭建个人博客的终极资料）
  • Hexo+GitHub Pages

简介

Hexo是一个快速、易用且高效的博客框架。Hexo使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

GitHub的好处自然不用多说，Github Pages 可以为项目建立静态主页，使用Hexo+GitHub Pages方案搭建个人博客时间成本和维护成本将大大降低。

准备

安装Hexo前需要准备以下环境

  • Git
  • Node.js（很简单，这里就不赘述了，详细过程可参考博文和GitHub）

安装

使用Node.js自带的npm全局安装Hexo，打开Git Bash Here输入

$ npm install -g hexo-cli

输入$ hexo -v验证安装是否成功

在以上安装过程中可同步进行GitHub的准备工作

创建SSH Key

打开Git Bash Here，输入

$ ssh-keygen -t rsa -C "你的邮箱地址（例：xxx123@gamil.com）"

完成之后在用户目录中找到.ssh目录，不出意外会有id_rsa和id_rsa.pub两个文件

id_rsa是私钥，自己妥善保管；id_rsa.pub是公钥，后面将会用到。

登录GitHub，点击Settings —> SSH and GPG keys —> New SSH Key，Title任意，在Key框内粘贴id_rsa.pub内容，点击Add SSH Key完成创建。

新建项目

点击new repository，在Repository name输入框中填写username（你的账户名）.github.io，如unlzh.github.io

创建项目成功后，访问https://你的账户名.github.io也就是刚刚填写的Repository name，出现项目名称等字样就说明成功。

开始搭建

初始化Hexo

Hexo安装成功后，输入

$ hexo init <folder>

即为你指定的文件夹，如何没有设置，默认在当前文件夹中建立项目

进入项目目录并安装依赖

$ cd <folder>

$ npm install

完成后执行$ hexo server启动服务器，访问http://localhost:4000，就可以看到hexo默认的界面
