主题

初步搭建好博客之后，访问自己的博客地址会出现Hexo默认的主题界面。相信这默认界面不会讨大多数人欢心吧。别担心，Hexo支持配置其他主题。

前往Hexo主题官网选择自己心仪的主题，博主使用的主题是Melody。选择好之后clone到/themes，然后在站点配置文件_config.yml中将主题修改为刚刚clone下来的主题：

这样就可以实现主题切换了，是不是很简单！

一般每个主题都有详细的说明文档，根据文档配置即可。

有些主题需要安装一些插件，参照其README操作。

图床

博客需要存储很多图片，例如每篇博文的配图，banner等。这些图片也可以存储在本地文件夹中，打开博客时解析生成，但是当需要的图片越来越多时，访问就会变得十分卡慢。因此最好的做法是将图片上传至图床，由图床生成链接来调用。国内的新浪图床、七牛云都是不错的选择。博主使用的是腾讯云的对象存储，正值腾讯云活动，oss对象存储免费使用50G容量，每月读写请求各100万次，用来做图床非常不错。

下面简单介绍使用腾讯云对象存储作图床的流程。

  1. 登录腾讯云（一般微信扫码就可以登录）
  2. 在免费产品中选择oss对象存储，开通
  3. 新建存储桶（bucket），注意设置存储桶访问权限（作图床的话一般设置为“公有读私有写”）和防盗链（在基础配置栏，防盗链设置将博客地址添加进白名单）
  4. 可下载官方客户端工具COS Browser，简洁好用，也可以基于API自己开发应用
  5. 将图片上传（最好将图片分类归档好，方便管理），取得链接

许多主题自带了第三方组件，配置即用，十分方便。例如：评论系统。博主使用的是Gitalk。

要想使用Gitalk，首先需要申请GitHub Application，获得clientID和clientSecret。

可能会出现的问题：

  • Gitalk组件上方出现“Error:not found”，检查填入的配置参数是否有误，尤其是“repo”（填的是博客在GitHub仓库的名称，例如我的是unlzh.github.io，这个很容易跟博客地址混淆）
  • “未找到相关的Issue，请联系@XXX创建”，这是由于博客仓库中还没有创建相关Issue，在评论组件中点击“使用GitHub账号登录”，系统会自动初始化
  • “Error:Validation Failed”，这是GitHub限制labal长度不能超过50引起的，解决方案来自Issue115，可通过MD5加密解码后的location.pathname来缩短长度。这些可在Gitalk的相关Issue中找到解决方法

此外，还有其他组件，例如UV和PV统计、RSS订阅等等，根据主题支持程度不同而不同，需根据自己主题的文档进行相应的配置，动手能力强的也可以修改源码来添加。这些组件都是比较常见的，网上也能找到许多教程说明。一些组件需要Hexo中安装才能使用，根据提示操作即可。

SEO优化

要想让其他小伙伴在网上能搜到自己的网站，就需要进行SEO优化。对于hexo生成的博客来说，主要是要多用心去给各个搜索引擎提交你的sitemap，让搜索引擎的爬虫多来爬你的站点，这样别人在搜索东西的时候才有更多的可能搜出你的文章，给你的博客带来人气。

下面就以谷歌收录为例。（不得不说，谷歌收录的效率比某度高太多…）

在谷歌搜索中输入site:<你的博客地址>，例如site:unlzh.github.io，若“没有找到相关网页”，就说明谷歌还没有收录你的博客。

新建robots.txt

Robots协议（也称为爬虫协议、机器人协议等）的全称是“网络爬虫排除标准”（Robots Exclusion Protocol），网站通过Robots协议告诉搜索引擎哪些页面可以抓取，哪些页面不能抓取。这个协议也不是一个规范，而只是约定俗成的，有些搜索引擎会遵守这一规范，有些则不然。通常搜索引擎会识别这个元数据，不索引这个页面，以及这个页面的链出页面。
